## Description / Justification

This server is being provisioned as a new UAT environment for the migration and upgrade of the existing Nexus Repository production instance (v3.64 → 3.93.2). Upon successful validation and user acceptance testing, the same environment will be promoted to Production.

The current production repository manages approximately **3 TB of artifact data**, requiring a dedicated filesystem for Nexus blob storage. The proposed design allocates `/u01` as the primary blob store with an initial capacity of **4 TB**, expandable to **8 TB** to support future repository growth.

The operating system and application components are distributed across dedicated logical volumes to align with current production standards and ensure operational consistency:

* `/` – OS (100 GB)
* `/home`, `/tmp`, `/var`, `/var/log`, `/var/log/audit` – system separation for stability and logging isolation
* `/u01` – Nexus blob store (4 TB)

Nexus Repository 3.93.2 will utilize an external **PostgreSQL database** for metadata storage (components, assets, users, and configuration), while artifact storage remains on the `/u01` filesystem.

The specification includes additional CPU (8 vCPU) and memory (16 GB RAM) to support migration activities (H2 export, indexing rebuilds, and repository cutover) and ensure stable production performance under increased load.
