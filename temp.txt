As part of the Bitbucket Disaster Recovery (DR) setup and testing, firewall access is required from the Bitbucket DR server to the configured LDAP server.

The Bitbucket user directory is configured to use **LDAP over TCP port 389 with SSL/TLS disabled** (matching the current Production configuration).

Kindly open firewall access from the Bitbucket DR server to the LDAP server on **TCP/389** to enable LDAP authentication for the DR instance.
