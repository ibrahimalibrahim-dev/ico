Firewall Request – Bitbucket Production & DR Database Connectivity (TCP/1433)




As part of the Bitbucket Disaster Recovery (DR) validation, we investigated a startup issue on the newly cloned DR Bitbucket server. Although the Bitbucket service is running, the application fails to initialize and reports that the Spring WebApplicationContext is unavailable.

As part of the troubleshooting, the following checks were performed:

* Verified that the Bitbucket service is running on both Production and DR servers.
* Verified that no host-based firewall is blocking traffic on either server (`firewalld` is inactive and no `iptables` rules are configured).
* Performed TCP connectivity tests to the SQL Server databases on port 1433.

Connectivity test results:

* Production Bitbucket → Production Database: **Successful**
* DR Bitbucket → Production Database: **Timed out**
* Production Bitbucket → DR Database: **Timed out**
* DR Bitbucket → DR Database: **Timed out**

Based on these results, the issue appears to be network connectivity rather than an operating system or application configuration issue.

Kindly implement the required firewall rules to allow SQL Server connectivity (TCP/1433) between the Bitbucket Production and DR servers and the corresponding Production and DR database servers, as per the approved DR architecture.

This connectivity is required to complete Bitbucket DR validation and support database connectivity during both normal operation and disaster recovery scenarios.
